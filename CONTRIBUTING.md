# Contributing to Astro-Vysio

Thank you for your interest in contributing to Astro-Vysio! This document provides guidelines and information for contributing to the project.

## 🎨 Share Your Creations

We love seeing what you create with Astro-Vysio! Here's how to share:

1. **Open an Issue** with the tag `[Showcase]` and include:
   - Screenshots or videos of your visualization
   - The audio track you used (if you're comfortable sharing)
   - Any custom presets or settings that made it special
   - A brief description of your creative process

2. **Social Media**: Tag us when you post your creations!

## 🐛 Reporting Bugs

Found a bug? Help us fix it!

1. Check if the issue already exists in the [Issues](https://github.com/kajica2/Astro-vyzor-main/issues) page
2. If not, create a new issue with:
   - A clear, descriptive title
   - Steps to reproduce the bug
   - Expected vs. actual behavior
   - Browser/OS information
   - Screenshots or error messages if applicable

## 💡 Suggesting Features

Have an idea? We'd love to hear it!

1. Check existing issues to see if it's already been suggested
2. Open a new issue with the `[Feature Request]` tag
3. Describe:
   - What you'd like to see
   - Why it would be useful
   - How it might work (if you have ideas)

## 🔧 Code Contributions

### Getting Started

1. **Fork the repository**
2. **Clone your fork**:
   ```bash
   git clone https://github.com/YOUR_USERNAME/Astro-vyzor-main.git
   cd Astro-vyzor-main
   ```
3. **Install dependencies**:
   ```bash
   npm install
   ```
4. **Create a branch** for your changes:
   ```bash
   git checkout -b feature/your-feature-name
   ```

### Development Guidelines

- Follow the existing code style and patterns
- Write clear, descriptive commit messages
- Add comments for complex logic
- Test your changes thoroughly
- Update documentation if needed

### Submitting Changes

1. **Commit your changes**:
   ```bash
   git commit -m "Add: descriptive message about your change"
   ```

2. **Push to your fork**:
   ```bash
   git push origin feature/your-feature-name
   ```

3. **Open a Pull Request**:
   - Provide a clear description of what you changed and why
   - Reference any related issues
   - Include screenshots if your changes affect the UI

## 🎯 Priority Areas

We're especially looking for help with:

- **Vector Graphics Support**: SVG-based visualizations and effects
- **3D Visualization Engine**: Three.js integration and 3D effects
- **Performance Optimization**: Making the app faster and smoother
- **Documentation**: Improving guides and examples
- **Testing**: Adding unit and integration tests
- **Accessibility**: Making the app more accessible to all users

## 📋 Code of Conduct

- Be respectful and inclusive
- Welcome newcomers and help them learn
- Focus on constructive feedback
- Celebrate diverse perspectives and ideas

## ❓ Questions?

Not sure where to start? Feel free to:
- Open an issue with the `[Question]` tag
- Check existing documentation
- Look at open issues labeled `[Good First Issue]`

Thank you for helping make Astro-Vysio better! 🚀
